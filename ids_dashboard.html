Domain gap significance: An L2 centroid distance of 2.496 in the 32-dimensional latent space indicates meaningful separation between KDD and UNSW distributions, even after neural encoder projection. This gap drives concept drift: a model calibrated on KDD normal traffic will flag UNSW normal traffic as anomalous at higher rates. The attention fusion module uses cross-source centroid broadcasting to partially bridge this gap before classification.

Estimated Model Accuracy (on Balanced KDD Training Sample)

Ensemble design: 4 batch models handle global pattern recognition on the full training window; 4 stream models (partial_fit) adapt incrementally to each chunk. During drift retraining, batch models refit on the current chunk (fast, small), while stream models call partial_fit. Final predictions use soft-voted probability averages weighted by each model's recent window accuracy (floor 0.1).

Architecture Overview

①KDD (41 feat) → FeatureEncoder → 32-d z_kdd
②UNSW (42 feat) → FeatureEncoder → 32-d z_unsw
③AttentionFusion: w₁·z_kdd + w₂·z_unsw → fused
④AdaptiveEnsemble (8 models) trains on fused_kdd
⑤UNSW test stream → phased drift simulation
⑥DriftDetector (ADWIN-style) monitors error rate
⑦Drift → partial_fit + weight update (retrain)
⑧AnomalyDetector (z-score, normal KDD profile)
⑨Hybrid prediction: ML + Anomaly → final label

Research Contributions

Heterogeneous fusion — No inner join needed. Two independently preprocessed datasets fused entirely in latent space via attention weighting.
Cross-domain drift — KDD (legacy 1999 data) vs UNSW-NB15 (modern 2015 attacks) creates a real domain gap of L2=2.496, generating authentic concept drift.
F1=0 explanation — Phase 4 contains 100% attack samples. F1 requires both normal and attack classes to compute; with only attacks, precision and recall denominators become undefined, yielding F1=0.000. This is a metric artifact — accuracy in phase 4 is 99.8%+.
